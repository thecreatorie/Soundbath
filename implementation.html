Tool 01 · First thing, Week 1
Cloudflare
DNS, CDN, security, and domain routing — the foundation everything else points through
Phase 1 Week 1 · Day 1 Free plan
What it is

Cloudflare is a DNS provider, CDN, and web security platform. When you move a domain to Cloudflare, all traffic to that domain routes through their global network — giving you fast DNS propagation, DDoS protection, SSL certificate management, and the ability to add page rules, redirects, and caching with no server required.

What we're using it for

Managing the DNS records for ThisIsSoundbath.com (and any acquired domains). Cloudflare sits in front of everything — it points the root domain to Framer, routes email subdomains to Google Workspace, and ensures all web traffic is HTTPS with auto-renewing SSL. Every other tool in this stack that needs a DNS record (HubSpot email verification, Beehiiv custom domain, Framer publish) gets configured here.

Other capabilities (not using now)
DNS management SSL / HTTPS Domain routing to Framer Email MX records DDoS protection Page caching rules Cloudflare Workers (serverless) Access (zero-trust auth) R2 storage Web Analytics (free, cookieless)
Why it's the right choice

Free tier covers everything SOUNDBATH needs. DNS propagation is near-instant (vs. 24–72 hours with most registrars). The Framer ↔ Cloudflare integration is one DNS record and done. Every major tool in this stack (Google Workspace, HubSpot, Beehiiv) has documented Cloudflare setup guides. It's also the only DNS provider where you can add page rules, redirects, and sub-domain routing without touching a server.

How to implement
  1. 1
    Create a Cloudflare account at cloudflare.com using the SOUNDBATH business email (create this first in Google Workspace — see Tool 02). Add the site by entering thisIssoundbath.com.
  2. 2
    Select the Free plan. Cloudflare will scan existing DNS records from the current registrar and import them automatically. Review the list — keep any existing records, mark anything unknown for follow-up.
  3. 3
    Update nameservers at the registrar (GoDaddy / Namecheap / wherever Vishal registered the domain). Cloudflare will give you two nameserver addresses like elsa.ns.cloudflare.com. Log into the registrar, find DNS / Nameserver settings, replace existing nameservers with Cloudflare's two. Save. Propagation takes 5 minutes to 24 hours — usually under 30 minutes.
  4. 4
    Enable "Always Use HTTPS" under SSL/TLS → Edge Certificates. Turn on "Automatic HTTPS Rewrites" too. This ensures every page is served securely with no mixed-content warnings.
  5. 5
    Add a CNAME record for Framer (do this when Framer is set up — see Tool 05). Framer will give you a CNAME target. Add it here under DNS → Add record → Type: CNAME, Name: @ or www, Target: Framer's provided URL.
  6. 6
    Add email records for Google Workspace (do this when Workspace is provisioned — see Tool 02). Google will give you 5 MX records and a TXT record for domain verification. Add them all to Cloudflare DNS.
Tool 02 · Week 1, Day 1–2
Google Workspace
Business email, shared calendar, Drive, and the account foundation every other tool logs into
Phase 1 Week 1 · Before all other accounts $7/user/mo Starter
What it is

Google Workspace is Google's business productivity suite — Gmail, Calendar, Drive, Docs, Meet, and admin controls — running on a custom domain. It's how hello@thisIssoundbath.com becomes a real business email rather than a Gmail alias.

What we're using it for

Creating the business email infrastructure that everything else depends on. Every tool in this stack — HubSpot, Beehiiv, Framer, Glofox, Circle.so — requires a valid business email for account creation and email sending. No other accounts should be created until Workspace is live. Also used for shared project Drive folder between you and Vishal, and Shared Calendar for project milestones and launch dates.

Other capabilities (not using now)
hello@thisIssoundbath.com Admin account + team aliases Shared Drive folder Shared calendar Google Meet Docs / Sheets / Forms Vault (email archiving) AppSheet (no-code apps)
Why it's the right choice

Google Workspace is the default business email infrastructure for a reason — every tool in this stack has a native Google login option, which means Vishal and the team can SSO into most platforms without creating new passwords. Gmail's spam reputation means transactional emails from HubSpot and Beehiiv verified on this domain will land in inboxes. At $7/user/month for the Starter tier, it's the cheapest professional email option available.

How to implement
  1. 1
    Go to workspace.google.com and click "Get started." Enter the business name (SOUNDBATH), number of employees, and country. At the domain step, select "I have a domain" and enter thisIssoundbath.com.
  2. 2
    Create the admin account: hello@thisIssoundbath.com. This is the master account — Vishal should own the billing, but you need admin access during setup. Use a strong password and store it in a shared 1Password or Bitwarden vault with Vishal.
  3. 3
    Verify domain ownership. Google will give you a TXT record to add to DNS. In Cloudflare, go to DNS → Add Record → Type: TXT, Name: @, Value: paste Google's string. Google verifies within minutes.
  4. 4
    Add MX records. Google gives you 5 MX records — add all of them to Cloudflare DNS. This routes incoming email to Gmail. Priority order matters: add them exactly as Google specifies.
  5. 5
    Create team aliases: info@, members@, bookings@. These forward to the admin account. Aliases cost nothing — only user seats cost $7/mo. Vishal only needs one seat initially; add more as staff are hired.
  6. 6
    Add you as a secondary admin. In the Admin Console → Users → Add new user, create dev@thisIssoundbath.com or similar. This gives you a working login for all setup work without using Vishal's master credentials.
  7. 7
    Create a shared Drive folder called "SOUNDBATH Project" and share it with both accounts. All brand assets, contracts, strategy docs, and Loom walkthroughs go here.
Tool 03 · Week 2 — Set up BEFORE any traffic
HubSpot CRM
The central contact record. Every guest, member, and lead — one source of truth from day one
Phase 1 Week 2 · Before Beehiiv goes live Free CRM · $15/seat Starter
What it is

HubSpot is a CRM (Customer Relationship Management) platform. At its core it's a database of contacts — every person who has interacted with SOUNDBATH, their history, where they came from, and what they've done. It also includes email marketing, forms, pipeline management, and automation. The free tier is genuinely powerful; Starter unlocks custom properties and removes HubSpot branding from emails.

What we're using it for

Source of truth for every contact. When someone subscribes to Beehiiv, books through Glofox, attends a Luma event, or fills in a Framer form — HubSpot is where that person's full record lives. The critical setup work is building the custom contact properties schema before any traffic comes in. If you add HubSpot after Beehiiv is live, you lose source attribution on early subscribers and can never recover it.

Data flows into HubSpot from
Beehiiv subscriber→Zapier webhook→HubSpot contact
Framer form→HubSpot native form embed→HubSpot contact
Glofox new member→Zapier→HubSpot contact · glofox_member_id
Luma attendee→Zapier→HubSpot contact · event source
Other capabilities (not using now)
Contact database + properties Source attribution Lifecycle stage tracking Form embeds for Framer Zapier integration Email sequences Deal pipelines (sales CRM) Landing pages Chatbot SMS (Starter+) Reporting dashboards
Why it's the right choice

Free tier is the most capable free CRM on the market — no contact limits, no time limit. The native Zapier and Beehiiv integrations mean almost no custom code. Every competitor in the peer set has fragmented contact data; HubSpot gives SOUNDBATH a unified record from subscriber zero. When it's time to do lifecycle email marketing (post-launch), HubSpot Starter email sequences are already set up and the contact data is clean. No migration cost later.

How to implement
  1. 1
    Create HubSpot account at hubspot.com using hello@thisIssoundbath.com. Select "Marketing" as the primary use case. Free tier is fine to start.
  2. 2
    Build the custom contact properties schema. Go to Settings → Properties → Create property for each of these fields: source (dropdown: Beehiiv / Framer form / Luma / Glofox / Referral / Direct), founding_member (checkbox: true/false), glofox_member_id (single-line text), membership_tier (dropdown: Founding / Standard / Premium — adjust to match Vishal's tier names), referral_code (single-line text), referral_source (single-line text — tracks who referred them).
  3. 3
    Set up lifecycle stages. In the default lifecycle stage field, rename or map stages to: Subscriber → Waitlisted → Founding Member → Active Member → Lapsed. This becomes the backbone of every HubSpot list and automation.
  4. 4
    Create a HubSpot form for the Framer site. Go to Marketing → Forms → Create form. Fields: First name, Last name, Email, "How did you hear about SOUNDBATH?" (maps to source). Copy the embed code — you'll paste it into Framer's custom code block.
  5. 5
    Connect your domain for email sending. Go to Settings → Email → Connect Domain. HubSpot will give you SPF and DKIM DNS records. Add them to Cloudflare. This ensures HubSpot emails sent from hello@thisIssoundbath.com pass spam filters.
  6. 6
    Create a "Founding Members" smart list. Go to Contacts → Lists → Create list → Active list. Filter: founding_member is true. This list auto-updates as new founding members are added from any source — Beehiiv, Framer form, Glofox, Zapier. It becomes the audience for every founding member email campaign.
  7. 7
    Upgrade to Starter ($15/seat/mo) when ready to send email sequences. Free tier handles all contact management and forms. Starter is only needed when you want to send automated email sequences from HubSpot directly (post-launch retention campaigns). Hold off until the list has 500+ contacts.
Tool 04 · Week 2–3 — The founding member engine
Beehiiv
Newsletter platform built for growth — referrals, subscriber analytics, and a publication-quality send experience
Phase 1 Week 2–3 Free to 2,500 · $43/mo Scale (annual)
What it is

Beehiiv is a newsletter platform built specifically for audience growth — not just email sending. It has native referral mechanics (readers get a personal referral link; you set a reward threshold), subscriber analytics at the individual level, a publication-style post editor, and a custom domain for the web view. Think of it as Substack with better growth tools and no revenue sharing.

What we're using it for

The pre-launch founding member pipeline. Beehiiv is live before the Framer site is fully built. It's the first thing that goes public — a signup page where interested people can join the SOUNDBATH founding member list and get a referral link. Every subscriber can share their link; when 3 friends subscribe (the threshold you set), they unlock the founding member offer. This creates a viral loop before a single brick is laid. Zapier connects each new subscriber to HubSpot with source: Beehiiv and founding_member: true (if referral threshold met).

The welcome sequence — 4 emails, set up before launch

Email 1 (immediate): "You're on the list." Confirm the subscription, introduce SOUNDBATH in 3 sentences, share the referral link, explain the founding member offer. Keep it short. First impression.

Email 2 (Day 3): "Why Miami, why now." The story of SOUNDBATH — what gap it fills, why the social wellness moment is here, what it will feel like to be a member. This is Vishal's voice. Pull directly from his one-paragraph brief.

Email 3 (Day 7): "What founding members get." Concrete and specific — the tier, the price lock, the early access, the community. This is the email that converts curious subscribers into committed founding members. Don't send this until the founding member offer is fully defined.

Email 4 (Day 14): "Refer a friend." Push on the referral mechanic specifically. Show them how many referrals they've made. Remind them of the reward. Make it easy with a pre-written social caption they can copy.

Other capabilities (not using now)
Newsletter publication Referral program (native) Welcome sequence automation Custom domain Subscriber analytics Zapier / HubSpot integration Paid subscriptions (monetize newsletter) Boosts (pay to grow subscriber list) Ad network Web archive of past issues Audience segments
How to implement
  1. 1
    Create Beehiiv account at beehiiv.com using hello@thisIssoundbath.com. Publication name: SOUNDBATH. Start on the free plan — you'll upgrade to Scale when the list hits ~2,000 or when you want to activate the referral program (referrals are a Scale feature).
  2. 2
    Connect the custom domain. In Beehiiv Settings → Domain, set the web publication to letters.thisIssoundbath.com or founding.thisIssoundbath.com. Beehiiv gives you a CNAME record — add it to Cloudflare DNS. The subscribe page will live at this URL.
  3. 3
    Set up sender authentication. Beehiiv will ask you to add SPF and DKIM records so emails send from hello@thisIssoundbath.com. Add these to Cloudflare. Without this step, emails may land in spam.
  4. 4
    Build the subscribe page. Under Grow → Subscribe Page, upload the SOUNDBATH logo, write a 2-sentence description, and set the headline. This page goes live before the Framer site is built — it's the first thing you send traffic to. Keep it simple: what is SOUNDBATH, why join, what founding members get.
  5. 5
    Build the 4-email welcome sequence. Under Automations → Create Automation → Trigger: New subscriber. Add 4 email steps with the delays described above. Write Email 1 immediately. Hold Emails 2–4 until Vishal has approved the founding member offer copy.
  6. 6
    Configure the referral program (after upgrading to Scale). Under Grow → Referral Program → Enable. Set reward threshold: 3 referrals = founding member status. Set the reward: a custom message confirming their founding member status, linked to a HubSpot form to capture their details. Beehiiv generates a unique referral link for every subscriber automatically.
  7. 7
    Connect Beehiiv to HubSpot via Zapier. Trigger: New subscriber in Beehiiv. Action: Create or update contact in HubSpot with source: Beehiiv, lifecycle_stage: Subscriber, referral_code (from Beehiiv's subscriber data). Test with a real email subscription before going live.
Critical timing note

Don't launch Beehiiv before HubSpot is set up and the Zapier connection is tested. Early subscribers are the most valuable contacts — if HubSpot isn't live and properly receiving source data, you lose attribution on everyone who signed up before the connection was established. You cannot retroactively assign source data in HubSpot.

Also: the referral program is a Scale-only feature. Free tier has a 2,500 subscriber limit and no referral mechanics. Budget for the upgrade to Scale ($43/mo annual) from Day 1 — the referral program is worth it.

Tool 05 · Week 3–4 — Brand site and soft launch page
Framer
React-output no-code site builder with CMS — brand-first, fast to publish, and genuinely premium
Phase 1 Week 3–4 $30/mo Pro (annual) — verify current
What it is

Framer is a no-code website builder that outputs clean React — meaning the published site is fast, SEO-friendly, and not tied to template constraints. It has a visual canvas editor, built-in CMS for dynamic content (blog posts, event listings, team bios), custom code blocks for embedding third-party tools, and one-click publishing to a custom domain via Cloudflare.

What we're using it for

The SOUNDBATH brand site. Phase 1 delivery is a single-page soft-launch site: brand identity, what SOUNDBATH is, the founding member CTA, and a HubSpot form embed. This goes live in Week 4 and sits at thisIssoundbath.com. Phase 1 also includes building the full multi-page site in the same Framer project — About, Membership, Events, Community — which goes live when content and photography are ready.

Build order inside Framer

First publish — the soft launch page (Week 4): Logo, hero statement, 2–3 sentence brand description, founding member CTA, HubSpot form embed, Beehiiv subscribe link. This is what Beehiiv referral links point to. No nav, no footer, just the hook.

Second publish — full site (Phase 1 complete): Full navigation with About, Membership, Events, Book Now (Glofox embed in Phase 2), Community (Circle link in Phase 2). CMS for blog/journal posts configured so Vishal can add content without you.

Other capabilities (not using now)
Custom domain publish (Cloudflare) HubSpot form embed CMS for blog/journal Animations + scroll effects Mobile-responsive by default Framer AI (layout generation) E-commerce (Stripe native) Localization (multi-language) Framer Sites marketplace (templates) Password protection (built-in)
How to implement
  1. 1
    Create Framer account at framer.com using hello@thisIssoundbath.com. Start a new blank project — don't use a template, the brand design needs to start from scratch to match the SOUNDBATH identity.
  2. 2
    Set up the design system first. Before building a single page, define: color variables (cream background, dark ink, gold accent — pull from Vishal's brand guide), typography (import Google Fonts or upload custom fonts), spacing scale, and button styles. This takes 2–3 hours but saves time on every page built after it.
  3. 3
    Build and publish the soft launch page. Single canvas page: full-bleed brand imagery, one strong headline, 2-sentence description, founding member CTA button that opens/scrolls to the HubSpot form. Embed the HubSpot form using Framer's Custom Code block — paste HubSpot's embed script. Test form submission before publishing.
  4. 4
    Connect to custom domain. In Framer → Project Settings → Custom Domain, enter thisIssoundbath.com. Framer gives you a CNAME record — add it in Cloudflare. Takes 5–10 minutes to propagate. Enable "Redirect www to root" in Framer settings.
  5. 5
    Build out full site pages. After soft launch is live: create a Framer CMS collection for "Journal" (blog posts) and "Events." Build pages for About, Membership (with pricing tiers), Events listing, and a placeholder for Book Now (Glofox embed will be added in Phase 2). Use Framer's component system — build the nav and footer as reusable components so changing them once updates everywhere.
  6. 6
    Add GA4 tracking code. When GA4 is set up (Tool 07), paste the GA4 measurement ID tracking script into Framer → Project Settings → Custom Code → Head. This fires on every page automatically.
  7. 7
    Test on mobile before every publish. Framer's desktop canvas doesn't always reflect mobile layout accurately. Use the mobile preview mode AND a real device before publishing any update. Priority pages to test: homepage, membership page, and any page with an embedded form.
Important: verify Framer pricing before billing Vishal

Framer's pricing has changed. Current Pro plan is approximately $30/mo (annual), up from $20/mo previously. Confirm the current price at framer.com/pricing before quoting. Also check whether the domain connection is included in the plan tier you're presenting — some features require mini or Pro.

Tool 06 · Phase 1 ongoing — The automation glue
Zapier
No-code automation connecting every tool in the stack — events in one platform trigger actions in another
Phase 1 Week 2 onward ~$20/mo Starter
What it is

Zapier is a no-code integration platform. You create "Zaps" — automated workflows where a trigger event in one app causes an action in another. No API coding required. It connects 6,000+ apps and handles the data routing that would otherwise require custom backend development.

The Zaps to build, in priority order

Zap 1 — Beehiiv → HubSpot (build Week 2, before Beehiiv goes live): Trigger: New subscriber in Beehiiv. Action: Create or update contact in HubSpot. Map fields: email, first name, source = "Beehiiv," founding_member = true (if referral threshold met), referral_code = subscriber's Beehiiv referral ID.

Zap 2 — Framer form → HubSpot (build Week 3, when soft launch goes live): Framer's native HubSpot embed handles this without Zapier — use the embed where possible. Only use Zapier if you need to cross-post to a second system (e.g., also add the contact to a Beehiiv list).

Zap 3 — Glofox → HubSpot (build Phase 2, when Glofox is live): Trigger: New member in Glofox. Action: Update HubSpot contact — set lifecycle_stage: Active Member, glofox_member_id, membership_tier.

Zap 4 — Luma → HubSpot (build Phase 2, when events start): Trigger: New event registrant in Luma. Action: Create/update HubSpot contact, set source: Luma event, tag with the event name.

Zap 5 — HubSpot lifecycle change → Notification (optional): Trigger: Contact lifecycle changes to "Founding Member." Action: Send a Slack or email notification to Vishal. Good for early momentum visibility.

How to implement
  1. 1
    Create Zapier account at zapier.com using hello@thisIssoundbath.com. Free tier allows 5 Zaps (single-step). Starter plan (~$20/mo) allows multi-step Zaps and filters — needed for the Beehiiv → HubSpot flow with conditional logic.
  2. 2
    Build Zap 1 first (Beehiiv → HubSpot), before Beehiiv is made public. Connect Beehiiv account, select trigger "New Subscriber." Connect HubSpot, select action "Create or Update Contact." Map all fields carefully — email must be the matching key. Run a test with a real email address to confirm the HubSpot contact is created with all properties populated.
  3. 3
    Add a filter step to Zap 1 to tag founding members separately. After the trigger, add a Filter step: only continue if Beehiiv subscriber's referral count is ≥ 3 (or whatever the founding member threshold is). Then add two branches: one that sets founding_member: true, one that sets it to false. Multi-step Zaps require the Starter plan.
  4. 4
    Document every Zap. In Zapier, rename each Zap clearly: "Beehiiv → HubSpot New Subscriber" not "Zap 1." Add a description in the Zap settings. When something breaks in six months, clear naming saves hours of debugging.
Tool 07 · Install when Framer goes live
Google Analytics 4
Website and conversion analytics — who's visiting, where they come from, and what they do
Phase 1 Week 4 (with Framer launch) Free
What we're using it for

Tracking site traffic sources, page performance, form conversion rates, and referral link performance. Critical for understanding which Beehiiv referral links are driving actual site visits, which pages are losing people before they fill the form, and where founding member signups are coming from.

How to implement
  1. 1
    Create a GA4 property at analytics.google.com using the SOUNDBATH Google Workspace account. Create a new property, select Web, enter thisIssoundbath.com.
  2. 2
    Copy the Measurement ID (format: G-XXXXXXXXXX). Paste it into Framer → Project Settings → Integrations → Google Analytics. Framer handles the rest — no manual script embed needed.
  3. 3
    Set up conversion events. In GA4 → Admin → Events, mark "form_submit" and "page_view" on the founding member page as conversions. This makes it easy to see how many visitors are converting to subscribers.
  4. 4
    Connect to HubSpot (optional, Phase 2). HubSpot can pull GA4 data into the CRM. Useful when you want to see website behavior alongside contact records.
  5. 5
    Use UTM parameters on all Beehiiv referral links. When someone clicks a referral link, the URL should include ?utm_source=beehiiv&utm_medium=referral&utm_campaign=founding_member. GA4 will attribute those site visits correctly. Beehiiv allows custom referral link parameters in Settings.
